🚀 DEPLOY IN 4 STEPS

  Step 1: Prepare
  $ cd infrastructure/terraform
  $ cp terraform.tfvars.example terraform.tfvars

  Step 2: Initialize (one-time)
  $ ./deploy.sh production init

  Step 3: Preview changes
  $ ./deploy.sh production plan

  Step 4: Deploy!
  $ ./deploy.sh production apply

Result: LoopBridge running in loopbridge-developer account! ✨

⚡ QUICK COMMANDS

  Deploy to staging (development):
  $ ./deploy.sh staging apply

  Deploy to production (live):
  $ ./deploy.sh production apply

  Preview changes:
  $ ./deploy.sh production plan

  Get outputs (IPs, endpoints):
  $ ./deploy.sh production output

  Destroy resources (careful!):
  $ ./deploy.sh staging destroy

  ✅ Automatic instance sizing (t2.micro for staging, t3.medium for prod)

  ✅ Safe deployment script with confirmations
  ✅ Multi-environment support in one account
  ✅ Repository-agnostic (change github_repo in terraform.tfvars)

🔧 THE SINGLE VARIABLE

  LOOPBRIDGE_ENV=staging

    Deploys:
    • EC2 t2.micro ($9/month)
    • RDS db.t4g.micro ($15/month)
    • No CloudFront CDN
    • No backups
    • Named: loopbridge-staging

  LOOPBRIDGE_ENV=production

    Deploys:
    • EC2 t3.medium ($40/month)
    • RDS db.t4g.medium Multi-AZ ($80/month)
    • CloudFront CDN ($150/month)
    • Daily backups (7-day retention)
    • Named: loopbridge

  The deploy.sh script sets this automatically!
